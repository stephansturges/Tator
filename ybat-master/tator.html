    Detection Recipes
    
                        
    Build, calibrate, and export reusable Ensemble Detection Recipes (EDRs). Use the Label Images tab to apply saved recipes.
    
                        
    
                            What a Detection Recipe is: An Ensemble Detection Recipe combines a prepass and a calibration.
                            The prepass builds a broad candidate pool from detectors + SAM3. The calibration learns how to filter that pool into cleaner final detections.
                            Defaults favor recall first; the calibrator and policy settings handle precision later. If you are unsure, keep defaults and only change one knob at a time.

                                        Recipe Logic + Knob Map
                                        
    
                                            
    How a recipe is split
    
                                            
    A Detection Recipe has two parts. The prepass generates a high-recall candidate pool. The calibration scores those candidates, tunes thresholds, and keeps the settings that survive promotion gates.
    
                                            
    Deterministic execution order
    
                                            
      
                                                
    1. Context setup: load dataset labelmap + glossary, selected model overrides, selected detector runs, and recipe config.
      2. 
                                                
    2. Detector seeding: each enabled detector runs twice (full frame + SAHI windows).
      3. 
                                                
    3. SAM3 text expansion: class prompts run on full image, plus optional windowed text pass.
      4. 
                                                
    4. Dedupe A: same-class IoU merge + optional scoreless cleanup.
      5. 
                                                
    5. SAM3 similarity expansion: pick exemplars per class, run global similarity, then optional windowed similarity pass.
      6. 
                                                
    6. Dedupe B: re-merge after similarity and re-apply cleanup.
      7. 
                                                
    7. Optional precision tightening: classifier-based rejection (if enabled).
      8. 
                                                
    8. Optional calibration filter: apply selected XGBoost calibration job.
      9. 
                                                
    9. Finalize/export: emit detections, provenance, and traces.
      10. 
                                            
    
                                            
    What clustering preserves
    
                                            
    Cluster winners retain full provenance: source_list, score_by_source, and atom IDs. Calibration features and metric tiers consume this evidence directly.
    
                                            
    Similarity exemplar policy
    
                                            
    Per class: candidates must pass Min exemplar score. Strategy controls seed selection: Top score, Random sample (seeded), or Diverse (score + source stratification + spatial spread with fraction/min/max/source-quota knobs).
    
                                            
    Prepass knobs (this panel)
    
                                            
    Dataset + glossary: Dataset, Glossary source, Named glossary, Glossary text, Extend glossary with Qwen, Max new terms per class.
    
                                            
    Detectors: Enable YOLO, Enable RF-DETR, YOLO run override, RF-DETR run override, SAHI window size, SAHI overlap.
    
                                            
    SAM runtime: SAM model, SAM3 runtime score threshold, SAM3 mask threshold.
    
                                            
    SAM3 text: Enable SAM3 text, SAM3 text score threshold, Windowed SAM3 text, SAM3 text window mode, SAM3 text window size, SAM3 text window overlap.
    
                                            
    SAM3 similarity: Enable similarity, Similarity score threshold, Min exemplar score, Exemplar strategy, Exemplars per class (top/random), Seed (random/diverse), Diverse fraction/min/max/source quota, Windowed similarity extension, Similarity window mode, Similarity window size, Similarity window overlap.
    
                                            
    Dedupe + cleanup: Fusion mode, Dedupe IoU, Cross-class dedupe enabled + IoU (default off / 0.80), Scoreless min IoU.
    
                                            
    Precision controls: Tighten false positives, Detector confidence, Classifier min prob, Classifier margin, Classifier background margin, optional classifier and optional calibration model selector.
    
                                            
    Grid/overlay: Grid columns, Grid rows, Grid overlap.
    
                                            
    Caption metadata knobs: Caption profile, caption model, caption variant, caption max tokens (for trace/caption outputs).
    
                                            
    Calibration knobs (same tab, section below)
    
                                            
    Images to sample, Max SAM3 terms/class, base + relaxed FP/TP caps, recall floor, per-class thresholds, threshold steps, optimize metric, Label IoU, Eval IoU, Eval IoU grid, calibration dedupe IoU, dedupe IoU grid, Support IoU, fusion mode, cross-class dedupe controls.
    
                                            
    What the final calibrator does
    
                                            
    The final calibrator does not create boxes. It builds one feature row per candidate from source provenance, detector and SAM scores, agreement signals, geometry, and optional classifier features. XGBoost learns which candidates are likely real objects, then threshold tuning searches per-class cutoffs that satisfy the recall / false-positive targets while optimizing the selected metric, usually F1.
    
                                            
    Metric tiers: raw detector replay, post-prepass, post-cluster, post-XGB (all compared at IoU 0.50 by default).
    
                                            
    Heavy-job behavior: prepass/calibration are GPU-heavy backend tasks. While active, automation locks disable competing GPU tabs/actions until the job completes or is cancelled.
    
                                            
    Default baseline: detector conf 0.45, SAM3 text 0.20, similarity 0.30, exemplar min 0.60, SAM3 runtime 0.20/0.20, dedupe IoU 0.75, cross-class dedupe off, eval IoU 0.50, scoreless IoU 0.00.

                                Tuning tips:
    
                                • Lower LR scale (e.g., 0.05 or 0.02) to reduce early loss spikes.
    
                                • Meta’s default LR scale is 0.1; that makes transformer LR 8e-5, vision 2.5e-5, language 5e-6. Halving/dropping LR scale makes starts gentler.
    
                                • Warmup steps: start near-zero LR and ramp up over these steps; higher is gentler/safer, lower is hotter/risks spikes (try 20–100).
    
                                • Timescale: stretches the LR schedule; higher keeps LR steady longer, lower decays faster (try 20–40 for short runs).
    
                                • To simulate larger batches without extra VRAM, keep batch size at 1 and raise grad accumulation (e.g., 2–4).
    
                                • Typical recipe: resolution 1008, LR scale 0.05, grad accumulation 2, batch sizes 1/1, epochs 20.
                            
    
                        
    
                            Text prompt alignment (read this):
    
                            • SAM3 class scores come from text embeddings. If you leave variants blank, we use your raw class names as prompts.
    
                            • Dataset-specific class names with underscores or abbreviations often don’t match the base model. Add synonyms per class or unfreeze the language backbone (set a small LR, or uncheck “Freeze language”).
    
                            • Freezing the language encoder is only sensible if you stick to the base vocabulary and just fine-tune boxes/masks.
    
                            • COCO eval is class-ID based; bad prompt alignment ⇒ wrong class IDs ⇒ low AP even if boxes look reasonable.

    SAM3 Recipe Mining
    
			                        
    Automatically discover “recipes” that help SAM3 find each class in your dataset. A recipe is a portable zip (JSON + embedded pretrained CLIP head) you can reuse in the labeling panel.
    
			                        
    Recipe mining UI build: 2025-12-20 (flat layout)
    
			                        
    
			                            
    How recipe mining works
    
			                            
    
			                                Prerequisite: Train a CLIP head first (CLIP tab). Recipe mining uses that head to score detections and to auto-tune “cleanliness” (precision) thresholds. This is required.
			                            
    
			                            
    
			                                Defaults are recall-first (lower thresholds, more candidates). Raise cleanliness or thresholds when you need higher precision.
			                            
    
			                            
    
			                                
    Full flow + how to tune it
    
			                                
    High-level flow
    
			                                
    Dataset (YOLO/COCO) + trained CLIP head
        │
        ├─ Sample images (deterministic by seed)
        │
        ├─ Build prompt list per class
        │    - class name + your extra prompts
        │    - optional Qwen prompt expansion
        │    - base prompts (e.g., “object”, “small object”)
        │
        ├─ Text-candidate eval (sample, text-only)
        │    - run SAM3 text inference per prompt
        │    - build a candidate-threshold curve per prompt (coverage vs false positives)
        │
        ├─ Build a multi-step recipe (per class)
        │    - pick a short chain of prompt steps (usually 3–8)
        │    - each step has its own text threshold (from the prompt curve)
        │
        ├─ Run the full steps pipeline (sample)
        │    - for each step:
        │        1) text candidates → pick diverse expansion candidates
        │        2) similarity-based search (SAM3 visual expansion)
        │        3) de-dupe
        │
        ├─ CLIP head gating (sample)
        │    - crop each detection and score with the pretrained CLIP head
        │    - auto-tune min prob + (optional) margin to hit your “cleanliness” target
        │
        ├─ Background suppression (default on)
        │    - if a __bg_* class scores higher than the target, drop the detection
        │    - optional margin can make this stricter
        │
        └─ Save portable recipe zip
             - recipe.json (schema v2, multi-step)
             - clip_head/head.npz + meta.json (embedded head + tuned thresholds)
    
			                                
    
			                                    Why recipes may look “text-only”: in schema‑v2, the “similarity prompt” is SAM3’s visual stage (candidate boxes → expand similar masks/boxes). The recipe stores the text prompts + thresholds; it does not need a fixed crop-bank of exemplars because candidates are chosen per image at inference time.
			                                
    
			                                
    What we optimize for (“cleanliness”)
    
			                                
    
			                                    We optimize to maximize coverage (find as many GT objects as possible) while meeting your precision target:
			                                    
      
			                                        
    • Match = a detection overlaps an unmatched GT box for that class with IoU ≥ “Eval IoU”.
      • 
			                                        
    • Precision = matches / (matches + false positives). Higher = fewer wrong boxes.
      • 
			                                        
    • Coverage = matches / GT count. Higher = finds more objects.
      • 
			                                        
    • Duplicates = extra detections on the same GT object (tracked separately).
      • 
			                                    
    
			                                
    
			                                
    Multi-step tuning modes (what to enable)
    
			                                
      
			                                    
    • Default (no extra tuning): candidate curves → greedy prompt selection → full pipeline → CLIP thresholds tuned to match your cleanliness.
      • 
			                                    
    • Refine prompt subset: quick “try a few swaps” pass (text-candidate stage only; no extra full SAM runs). Use when the chosen prompts look “obviously wrong”.
      • 
			                                    
    • Tier‑1 grid search: small search over visual_expansion_score (SAM3) + candidates/step (shared across steps). Good first knob search if Global is off.
      • 
			                                    
    • Tier‑2 IoU search: small search over de‑dupe overlaps (IoU) (shared across steps). Helps duplicates / false positives.
      • 
			                                    
    • Global optimizer (best quality): tries many full recipe variants and keeps the best (slowest, most thorough).
      • 
			                                
    
			                                
    
			                                    Note: when the Global optimizer is on, Tier‑1 and Tier‑2 are ignored (Global already tunes those knobs).
			                                
    
			                                
    Global optimizer: what knobs it can change
    
			                                
    
			                                    A trial is a full per-class recipe (ordered steps + per-step settings). We create new trials by making small changes to the current best:
			                                    
      
			                                        
    • Prompt subset: add / drop / swap a step’s prompt.
      • 
			                                        
    • Per-step text threshold: how strict the text prompt must be before a box becomes a candidate.
      • 
			                                        
    • Per-step visual expansion score (SAM3) and candidates per image (how many candidates we expand).
      • 
			                                        
    • Per-step de-dupe overlap (IoU): candidate de-dupe and output de-dupe.
      • 
			                                        
    • Optional: step order and per-step max dets/image.
      • 
			                                    
    
			                                    Each trial is scored by running the real steps pipeline on the same (repeatable) subset of sampled images, then sweeping CLIP thresholds to match your cleanliness target.
			                                
    
			                                
    Breadth vs depth (simple mental model)
    
			                                
      
			                                    
    • Breadth (“exploration”): try more recipe ideas. Increase when results feel “stuck”.
      • 
			                                    
    • Depth (“reliability”): test each idea on more sampled images. Increase when results feel noisy or unstable.
      • 
			                                    
    • Rule of thumb: if you raise breadth a lot, add some depth too.
      • 
			                                
    
			                                
    Global optimizer controls (what they mean)
    
			                                
      
			                                    
    • Stage‑1/2/3 eval cap: how many sampled images we use per stage (bigger = more reliable, slower). Stage‑1 ranks quickly; Stage‑2/3 re-test the best candidates.
      • 
			                                    
    • Max trials: how many candidate recipe configs we consider per round (higher = more thorough, slower).
      • 
			                                    
    • Keep ratio: how aggressively we prune candidates each stage (0.5 keeps the top half).
      • 
			                                    
    • Rounds + Variants per round: how long we keep exploring around the current best recipe.
      • 
			                                    
    • Tune step order / Tune max dets/image: optional extra dimensions the optimizer is allowed to change.
      • 
			                                
    
			                                
    How to influence results (practical)
    
			                                
      
			                                    
    • Cleaner / fewer FPs: raise “Recipe cleanliness”. Expect lower coverage.
      • 
			                                    
    • More coverage: lower “Recipe cleanliness”, increase Max recipe steps, and consider lowering Candidate curve floor (lets per-step thresholds go below the base).
      • 
			                                    
    • More thorough search: set “Search budget” to Balanced/Best and increase Max trials / Rounds / Variants per round (slower).
      • 
			                                    
    • Faster runs: reduce “Eval images”; lower Stage‑3 eval cap; lower Max trials.
      • 
			                                
    
			                                
    
			                                    Note: we no longer generate the old crop-bank recipe style; new recipes are schema‑v2 multi-step and always embed a pretrained CLIP head. Imported legacy recipes can still be applied for backward compatibility.

    Recipe mining quick reference
    
			                            
    
			                            
    
			                                
    Workflow: Train a CLIP head → run mining (start with a small sample) → review per-class results → save recipes → apply (and optionally chain) recipes from the labeling tab.
    
			                                
    How to read scores:
    
			                                
      
			                                    
    • Coverage = matches / GT objects (higher is better).
      • 
			                                    
    • Precision = matches / (matches + false positives) (higher is cleaner).
      • 
			                                    
    • FPs = detections that match no ground-truth box for that class (lower is better).
      • 
			                                    
    • Duplicates = extra detections on the same GT object (lower is better; tracked separately).
      • 
			                                    
    • Det rate = % of sampled images that produced at least one detection (useful “is anything happening?” sanity check).
      • 
			                                
    
					                                
    If results look “dead”: verify your CLIP head includes the dataset class names, and try lowering “Candidate curve floor” or temporarily lowering “Recipe cleanliness” to confirm SAM3 is producing candidates before tightening filters.
    
			                                
    Knob glossary (all controls)
    
			                                
      
			                                    
    • Eval images: how many images are sampled for mining + scoring.
      • 
			                                    
    • Sample seed: deterministic sample selector (same seed = same sample).
      • 
			                                    
    • Reuse cached SAM results: reuse cached SAM3 outputs between runs.
      • 
			                                    
    • Match overlap (Eval IoU): IoU threshold for scoring matches only.
      • 
			                                    
    • Predictors per GPU: SAM3 workers per GPU (more = faster, more VRAM).
      • 
			                                    
    • Max predictors total: optional cap on total workers.
      • 
			                                    
    • Max recipe steps: max prompt steps per class.
      • 
			                                    
    • Similarity candidates per step: how many candidate boxes are expanded per image.
      • 
			                                    
    • Search budget: enables Global optimizer presets (Off/Fast/Balanced/Best).
      • 
			                                    
    • Compute estimate: relative cost indicator (not seconds).
      • 
			                                    
    • Early-stop: stop adding steps when coverage stalls or precision is too low.
      • 
			                                    
    • CLIP prompt prefilter: drop weak prompts before SAM3 (fast).
      • 
			                                    
    • Global optimizer (breadth): trials, rounds, variants, step changes.
      • 
			                                    
    • Global optimizer (depth): stage eval caps, keep ratio.
      • 
			                                    
    • Tier‑1 grid search: tunes visual expansion score + candidates/step.
      • 
			                                    
    • Tier‑2 IoU search: tunes candidate/output de‑dupe IoUs.
      • 
			                                    
    • Refine prompt subset: small add/drop/swap search on prompts.
      • 
			                                    
    • Candidate curve floor / max dets: text-candidate curve controls.
      • 
			                                    
    • Text threshold (candidates): score cutoff for text prompt detections.
      • 
			                                    
    • Visual expansion score (SAM3): score cutoff for similarity-based expansion.
      • 
			                                    
    • Candidate de‑dupe IoU: remove near-duplicate candidate boxes.
      • 
			                                    
    • Output de‑dupe IoU: remove near-duplicate final boxes.
      • 
			                                    
    • Mask threshold: polygon mask binarization (segmentation only).
      • 
			                                    
    • Max dets/image: safety cap on outputs per image.
      • 
			                                    
    • Min area (px²): drop tiny detections.
      • 
			                                    
    • Simplify ε: polygon simplification amount.
      • 
			                                    
    • Pretrained CLIP head: required classifier for mining.
      • 
			                                    
    • Auto-tune thresholds: per-class CLIP min prob/margin tuning.
      • 
			                                    
    • Recipe cleanliness: target precision for CLIP tuning.
      • 
			                                    
    • Manual head min prob / margin: used when auto‑tune is off.
      • 
			                                    
    • Class IDs: optional subset of classes to mine.
      • 
                                        
    • Extra text prompts/class: Qwen prompt expansion count.
      • 
                                        
    • Qwen max tokens: prompt generator output length cap.
      • 
			                                    
    • Extra prompts JSON: user-provided prompts by class.

    Default upload and payload limits
    
                    
    
                        Defaults are tuned for LAN use. You can override any of these on the backend by setting
                        environment variables (e.g. CLIP_TRAIN_UPLOAD_QUOTA_BYTES,
                        DATASET_ZIP_MAX_BYTES, AGENT_RECIPE_MAX_BYTES,

                        SAM_PRELOAD_MAX_BYTES).
                        If the backend is configured with lower limits, uploads over those caps will be rejected.
                    
    
                    
      
                        
    • Dataset zips (SAM3/Qwen): up to ~100GB zip, ~50GB per entry.
      • 
                        
    • CLIP/Qwen chunked uploads: ~10GB per file, ~100GB per job quota.
      • 
                        
    • Classifier/labelmap/general assets: ~10GB per file, ~100GB per uploads quota.
      • 
                        
    • Agent recipe zips: ~2GB total; crops capped (~1000 pngs / ~512MB total).
      • 
                        
    • Responses: max ~5000 detections / 2000 masks; SAM preload cache ~2GB.
      • 
                        
    • Agent mining cache: ~80GB default; TTL off by default (set AGENT_MINING_CACHE_TTL_HOURS to enable purge).
